# HydroModelsRastersExt 安装和配置说明

## 📋 概述

本扩展为 HydroModels 提供了完整的栅格数据处理能力，专门用于：

✅ 读取各种格式的 DEM（数字高程模型）数据  
✅ 自动执行 D8 流向分析  
✅ 生成流向矩阵作为 HydroRoute 的输入  
✅ 支持多种栅格数据的读取和重采样  
✅ 与 HydroModels 框架无缝集成

---

## 🔧 系统要求

- **Julia 版本**: 1.9 或更高
- **操作系统**: Windows, macOS, Linux
- **内存**: 建议 4GB 以上（处理大型 DEM 时需要更多）

---

## 📦 依赖库安装

### 核心依赖（必需）

#### 1. Rasters.jl ⭐⭐⭐

这是本扩展的核心依赖，提供栅格数据读写功能。

```julia
using Pkg
Pkg.add("Rasters")
```

**功能：**
- 读取 GeoTIFF, NetCDF, ASCII Grid 等格式
- 栅格数据重采样和裁剪
- 坐标系统支持
- 懒加载和高性能

**版本要求：** 0.11.x 或 0.12.x

---

### 可选依赖（推荐）

#### 2. ArchGDAL.jl ⭐⭐

提供更多栅格格式支持和高级 GIS 功能。

```julia
Pkg.add("ArchGDAL")
```

**功能：**
- 支持 100+ 种栅格和矢量格式
- GDAL 的完整 Julia 封装
- 坐标转换和投影

**适用场景：**
- 需要读取特殊格式的栅格数据
- 需要精细的坐标系统控制
- 需要栅格与矢量数据互操作

#### 3. Proj.jl ⭐

坐标系统转换和投影变换。

```julia
Pkg.add("Proj")
```

**适用场景：**
- DEM 和其他数据使用不同坐标系统
- 需要进行投影变换
- 大地测量计算

#### 4. Images.jl / ImageFiltering.jl

图像处理算法，可用于栅格数据处理。

```julia
Pkg.add(["Images", "ImageFiltering"])
```

**适用场景：**
- DEM 平滑和滤波
- 边缘检测
- 形态学操作

## 🚀 快速安装

### 方案 A：最小安装（仅核心功能）

```julia
using Pkg

# 安装 Rasters.jl
Pkg.add("Rasters")

# 确保 HydroModels 已安装
Pkg.add("HydroModels")  # 或 Pkg.develop(path="path/to/HydroModels")
```

### 方案 B：推荐安装（完整功能）

```julia
using Pkg

# 安装所有推荐依赖
packages = [
    "Rasters",
    "ArchGDAL",
    "Proj",
    "GeoInterface"
]

Pkg.add(packages)
Pkg.add("HydroModels")
```

### 方案 C：完整安装（包括辅助工具）

```julia
using Pkg

# 完整安装脚本
packages = [
    "Rasters",        # 核心：栅格数据处理
    "ArchGDAL",       # 更多格式支持
    "Proj",           # 坐标转换
    "GeoInterface",   # 统一地理接口
    "Images",         # 图像处理
    "ImageFiltering", # 滤波操作
    "NCDatasets"      # NetCDF 支持
]

Pkg.add(packages)
Pkg.add("HydroModels")
```

## 🐛 故障排除

### 问题 1：扩展未加载

**症状：**
```julia
ERROR: UndefVarError: compute_d8_flow_direction not defined
```

**解决方法：**
```julia
# 确保按正确顺序加载
using HydroModels
using Rasters

# 检查 Rasters 版本
using Pkg
Pkg.status("Rasters")  # 应该是 0.11.x 或 0.12.x
```

---

### 问题 2：GDAL 相关错误

**症状：**
```julia
ERROR: could not load library "libgdal"
```

**解决方法：**

**方案 A - 使用 Conda（推荐）：**
```julia
using Pkg, Conda
Pkg.add("Conda")
Conda.add("gdal")
```

**方案 B - 系统安装：**

- **Windows**: 下载 GDAL installer
- **macOS**: `brew install gdal`
- **Linux**: `sudo apt-get install gdal-bin libgdal-dev`

---

### 问题 3：版本不兼容

**症状：**
```julia
ERROR: Unsatisfiable requirements detected for package Rasters
```

**解决方法：**
```julia
# 更新所有包
using Pkg
Pkg.update()

# 如果仍有问题，重新安装
Pkg.rm("Rasters")
Pkg.add("Rasters")
```

---

### 问题 4：内存不足（大型 DEM）

**症状：**
程序崩溃或非常慢

**解决方法：**
```julia
# 使用懒加载
dem = Raster("large_dem.tif", lazy=true)

# 或分块处理
dem_chunk = dem[X(1:1000), Y(1:1000)]

# 或增加 Julia 堆大小
# 启动时: julia --heap-size-hint=8G
```

## 📊 性能优化建议

### 1. 使用多线程

启动 Julia 时：
```bash
julia --threads 8
```

在代码中：
```julia
using Base.Threads
@threads for i in 1:n
    # 并行处理
end
```

### 2. 预编译

```julia
using Pkg
Pkg.precompile()
```

### 3. 使用 GPU（高级）

```julia
using CUDA
# 将数据移到 GPU
dem_gpu = CuArray(dem.data)
```
