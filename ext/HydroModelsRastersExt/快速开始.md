# HydroModelsRastersExt 快速开始指南

欢迎使用 HydroModels 的栅格数据处理扩展！本指南将帮助您快速上手。

## 📦 安装

### 第一步：安装 Rasters.jl

```julia
using Pkg
Pkg.add("Rasters")
```

### 第二步：确认 HydroModels 已安装

```julia
# 如果还未安装 HydroModels
Pkg.add("HydroModels")

# 或从本地开发版本
Pkg.develop(path="/path/to/HydroModels")
```

### 第三步：加载扩展

```julia
using HydroModels
using Rasters  # 加载Rasters会自动激活扩展
```

扩展会自动加载，无需额外配置！

---

## 🚀 五分钟上手

### 示例 1：从文件读取 DEM 并计算流向

```julia
using HydroModels
using Rasters

# 读取 DEM 文件（支持 GeoTIFF 等格式）
dem = Raster("path/to/your/dem.tif")

# 计算 D8 流向
flow_direction = compute_d8_flow_direction(dem)

# 查看结果
println("流向矩阵大小: ", size(flow_direction))
```

### 示例 2：创建 HydroRoute 的聚合函数

```julia
# 提取有效像元位置
positions = extract_positions_from_raster(dem)

# 创建流向聚合函数
aggr_func = HydroModels.build_aggr_func(flow_direction, positions)

# 现在可以在 HydroRoute 中使用 aggr_func 了！
```

### 示例 3：匹配不同分辨率的栅格

```julia
# 读取降雨数据
rainfall = Raster("rainfall.tif")

# 将降雨数据重采样到 DEM 的分辨率
rainfall_matched = match_raster_to_dem(rainfall, dem, method=:bilinear)
```

## 📖 核心功能

### 1️⃣ `compute_d8_flow_direction(dem)`

**功能：** 计算 D8 流向矩阵

**输入：**
- `dem`: Raster 对象（DEM 数据）

**输出：**
- 流向矩阵（Int），使用 D8 编码

**D8 编码规则：**
```
   32  64  128
    ↖  ↑  ↗
16 ← • → 1
    ↙  ↓  ↘
   8   4   2
```

**示例：**
```julia
flow_dir = compute_d8_flow_direction(dem)
```

---

### 2️⃣ `extract_positions_from_raster(raster, valid_mask)`

**功能：** 提取有效像元的位置

**输入：**
- `raster`: Raster 对象
- `valid_mask`: 验证函数（可选，默认为 `x -> !ismissing(x)`）

**输出：**
- 位置列表 `Vector{Tuple{Int,Int}}`

**示例：**
```julia
# 提取所有非缺失值的位置
positions = extract_positions_from_raster(dem)

# 只提取高程大于 100m 的位置
high_positions = extract_positions_from_raster(
    dem, 
    x -> !ismissing(x) && x > 100.0
)
```

---

### 3️⃣ `match_raster_to_dem(raster, dem, method)`

**功能：** 将栅格重采样到 DEM 的分辨率和范围

**输入：**
- `raster`: 需要重采样的栅格
- `dem`: 参考 DEM
- `method`: 重采样方法（`:nearest`, `:bilinear`, `:cubic`）

**输出：**
- 重采样后的栅格

**示例：**
```julia
# 使用双线性插值
matched = match_raster_to_dem(landuse, dem, method=:bilinear)

# 使用最近邻（适合分类数据）
matched = match_raster_to_dem(soil_type, dem, method=:nearest)
```

---

### 4️⃣ `create_hydroroute_from_dem(dem, name)`

**功能：** 从 DEM 提取 HydroRoute 所需的所有组件

**输入：**
- `dem`: Raster 对象
- `name`: 路由名称（可选）

**输出：**
- NamedTuple，包含：
  - `flow_dir`: 流向矩阵
  - `positions`: 位置列表
  - `aggr_func`: 聚合函数
  - `hru_types`: HRU 类型

**示例：**
```julia
info = create_hydroroute_from_dem(dem, name=:my_watershed)

# 使用提取的信息构建 HydroRoute
@hydroroute :my_route begin
    fluxes = begin
        @hydroflux outflow ~ k * storage
    end
    dfluxes = begin
        @stateflux storage ~ inflow - outflow
    end
    hru_types = info.hru_types
    aggr_func = info.aggr_func
end
```

## 🔧 完整工作流程示例

```julia
using HydroModels
using Rasters
using ComponentArrays

# === 第一步：读取和预处理 DEM ===
dem = Raster("data/watershed_dem.tif")
println("DEM 大小: ", size(dem))

# === 第二步：计算流向 ===
flow_direction = compute_d8_flow_direction(dem)

# 统计流向分布
d8_codes = [1, 2, 4, 8, 16, 32, 64, 128]
d8_names = ["东", "东南", "南", "西南", "西", "西北", "北", "东北"]
for (code, name) in zip(d8_codes, d8_names)
    count = sum(flow_direction .== code)
    println("$name: $count 像元")
end

# === 第三步：准备 HydroRoute 组件 ===
positions = extract_positions_from_raster(dem)
aggr_func = HydroModels.build_aggr_func(flow_direction, positions)

# === 第四步：读取和匹配其他数据 ===
rainfall = Raster("data/rainfall.tif")
rainfall_matched = match_raster_to_dem(rainfall, dem)

# === 第五步：构建 HydroRoute ===
# （需要根据具体模型定义 fluxes 和 dfluxes）
hru_types = ones(Int, length(positions))  # 假设所有网格类型相同

# === 第六步：运行模型 ===
# 准备输入数据、参数等，然后运行...
```

## 📊 支持的文件格式

通过 Rasters.jl，扩展支持以下格式：

| 格式 | 扩展名 | 说明 |
|------|--------|------|
| GeoTIFF | `.tif`, `.tiff` | 最常用的栅格格式 ⭐ |
| NetCDF | `.nc` | 气象数据常用 |
| GRIB | `.grib`, `.grb` | 气象数据 |
| ASCII Grid | `.asc` | ArcGIS 格式 |
| HDF5 | `.h5`, `.hdf5` | 大数据集 |

## 🐛 常见问题

### 问题 1：`UndefVarError: compute_d8_flow_direction not defined`

**原因：** 扩展未正确加载

**解决：**
```julia
# 确保按正确顺序加载
using HydroModels
using Rasters  # 这会触发扩展加载
```

### 问题 2：`GDAL not found`

**原因：** GDAL 未安装

**解决：**
```julia
using Pkg, Conda
Pkg.add("Conda")
Conda.add("gdal")
```

### 问题 3：坐标系统不匹配

**原因：** DEM 和其他栅格使用不同的坐标系统

**解决：**
```julia
# 检查坐标系统
crs(dem)
crs(rainfall)

# 重投影
rainfall_reprojected = reproject(rainfall, crs=crs(dem))
```
